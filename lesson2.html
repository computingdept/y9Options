Do Now

Think back to Lesson 1. You compared Computer Science and IT. Both use step-by-step thinking.

1. A step-by-step plan a robot can follow.
Answer: Algorithm (CS)
2. The stages your class follows to collect survey data and make a chart.
Answer: Workflow (IT)

Answer: Algorithm (CS)
4. The steps your school uses to request, approve and publish a poster.
Answer: Workflow (IT)

What you’ll learn

By the end of this lesson, you will be able to:

  • Say what an algorithm and a workflow are.
  • Spot examples of each in real life and in school.
  • Choose which one you prefer and explain why.

How you’ll know you’ve got it

  • Write a short algorithm for a simple task.
  • Build a short workflow that others can follow.
  • Share which you liked more and give a reason.

Key Vocabulary

Algorithm — a clear set of steps a computer can follow.
Workflow — the order of tasks people and tools follow to get work done.
Decision — a yes/no choice in an algorithm or process.
Input — information that goes into a system.
Output — the result the system gives back.

CS Activity – Write an Algorithm

  1. Write pseudocode to make toast with butter.
  2. Add one decision (e.g. Is the toast burnt?).
  3. Test it with a partner: follow it exactly!
Show sample answer 
# Pseudocode
if bread_available = TRUE then
    put bread in toaster
    wait until toasted
    if toast_burnt = TRUE then
        make new toast
    else
        spread butter
        serve
end if

IT Activity – Build a Simple Workflow

  1. Create a short workflow to plan a club poster.
  2. Use Forms (survey), SharePoint/Drive (file), and Docs/Slides (poster).
  3. Share it with a partner to follow and tick each step.
  • Collect info → pick design → make draft → peer review → publish.
Show sample answer

Example 5-step workflow: Create survey in Forms → Share link and collect responses → Export responses to Sheets/Excel → Draft poster in Slides/Publisher using top 3 messages → Peer review and publish to VLE/SharePoint.
