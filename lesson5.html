What you’ll learn

By the end of this lesson, you will be able to:

  • Explain what a LAN is and name key devices (router, switch, access point).
  • Describe IP and MAC addresses and what a packet is.
  • Run simple network tests (ping / tracert) and record results clearly.

How you’ll know you’ve got it:

  • You can sketch a classroom star network and label the devices.
  • You can explain IP vs MAC in one sentence each.
  • Your test table shows sensible results and comments.

Key Vocabulary

LAN
Local Area Network (e.g., your classroom or school).
Router
Forwards data between different networks (school ↔ internet).
Switch
Connects devices inside one network and forwards frames.
Access Point (AP)
Lets wireless devices join the network (Wi-Fi).
IP Address
Number used to find a device on a network (like 192.168.1.23).
MAC Address
Unique hardware address of a network card.
Packet
Small chunk of data with addresses and content.
Bandwidth
How much data can move per second.
Latency
Delay (how long a packet takes to travel).

CS Activity – Sketch a Star Network for a Classroom

Draw a simple star network for a class of 6 computers, a teacher laptop, a printer, and the internet.

  1. Put a switch in the middle.
  2. Connect 6 student PCs + 1 teacher laptop + 1 printer to the switch.
  3. Connect the switch to the router, and the router to the internet cloud.
  4. Label example IPs (private range): 192.168.1.10–192.168.1.17 for devices, 192.168.1.1 for router.
  5. Write one sentence each for: IP address, MAC address, packet.
Item Sample Why
Topology Star – all devices connect to a central switch Easy to add devices; one cable fault doesn’t stop all
IP plan Router 192.168.1.1, PCs .10–.15, Teacher .16, Printer .17 Private addressing for a small LAN
IP address “Where to send data on the network.” Like a postal address for devices
MAC address “Hardware ID burned into the network card.” Used on the LAN by the switch
Packet “Small box of data with from/to info.” Networks move packets, not whole files at once

IT Activity – Run Simple Network Tests and Log Results

Use built-in tools to measure connection quality. Record results in a small table (Excel or a worksheet).

  1. Find your IP:
    • Windows: Open Command Prompt → type ipconfig
    • Mac: Open Terminal → type ifconfig (or check Wi-Fi details)
  2. Ping a site (4 packets):
    • Windows: ping bbc.co.uk -n 4
    • Mac: ping -c 4 bbc.co.uk
  3. Trace the route (optional):
    • Windows: tracert bbc.co.uk
    • Mac: traceroute bbc.co.uk
  4. Log your results in a table with columns: Host • Avg ms • Packet Loss • Notes.
  5. Write one sentence to explain the difference between bandwidth and latency.
Field Example Notes
Host bbc.co.uk Public website (stable target)
Avg ms 24 Lower is faster (lower latency)
Packet Loss 0% Good connection if no loss
Notes Classroom Wi-Fi, full signal Record anything that might affect results
Bandwidth vs Latency Bandwidth = how much; Latency = how long. Amount vs delay

Exit Ticket

  1. What does a router do?
  2. What is a packet?
  3. Which has lower numbers if the connection is quick: latency or bandwidth?

💡 Sample Answers

  • Router: Sends data between different networks.
  • Packet: A small chunk of data with from/to info.
  • Latency: Lower is faster (bandwidth is “how much”).
