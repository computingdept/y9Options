What You’ll Learn

This lesson introduces SQL (Structured Query Language) — the language used to talk to and manage tables. You’ll learn how to ask a database (lots of tables) questions (known as queries) and how to filter or sort the answers you get back.

  • Understand what SQL is and how it can retrieve information from a database.
  • Use key commands such as SELECT, WHERE and ORDER BY.
  • Explore an interactive environment on SQL Island to practise real SQL queries.
  • See how conditions, sorting and pattern matching affect the results you get.
  • Reflect on how databases make data easier to search, edit and analyse than spreadsheets.

Knowledge Recap

Before using SQL, it helps to remember what you already know about tables from earlier lessons.

  • A database stores information in tables made up of rows (records) and columns (fields).
  • SQL (Structured Query Language) allows you to ask questions of that table using commands such as:
    • SELECT — choose which fields to view
    • FROM — choose which table to use
    • WHERE — filter results that match a condition
    • ORDER BY — sort the data
Analogy: Imagine you’re in McDonald’s or Nando’s. The restaurant’s menu is like a database table — each dish is a record, and each column (price, spice level, calories, ingredients) is a field. When you ask, “Show me all chicken meals under £8,” you’re doing exactly what SQL does: SELECT chicken dishes FROM menu WHERE price < 8 ORDER BY spice level. SQL is the waiter’s notepad language — it translates your request into clear instructions the kitchen (the database engine) can follow.
Key Idea: SQL turns real-world questions — like what to order or what data to find — into structured, logical instructions that computers can carry out instantly.

Key Vocabulary

All the data about one thing (one row).

One piece of data in a record (a column).

CS Activity – Explore SQL with SQL Island

Use the interactive world of SQL Island to learn how SQL queries work. Follow the on-screen instructions to complete each challenge and escape the island by writing SQL statements that retrieve, filter and sort data.

  1. Work through at least 6 – 8 stages on SQL Island.
  2. Keep a record of each SELECT statement you used and what it achieved.
  3. Try out WHERE, AND, OR, LIKE and ORDER BY in your own queries.
  4. When you finish, explain which commands helped you most and why.
Example Query What it Does Skill Shown
SELECT * FROM citizens; Displays every column and record in the citizens table. Retrieve all data from one table.
SELECT name, age FROM citizens WHERE age >= 18; Shows the names and ages of all adults. Filtering records with a condition (WHERE).
SELECT name, job FROM citizens WHERE job = 'sailor' OR job = 'guard'; Lists everyone who is either a sailor or a guard. Combining conditions with OR.
SELECT name FROM citizens WHERE name LIKE 'A%'; Finds citizens whose names start with the letter A. Pattern matching using LIKE.
SELECT name, age FROM citizens WHERE age >= 18 ORDER BY age DESC; Shows adults from oldest to youngest. Sorting results with ORDER BY.

IT Activity – Track SQL Island Progress in a Spreadsheet

Create a spreadsheet workbook with a separate tab (sheet) for each game stage: Task 1, Task 2, Task 3 … (or name by the island step). On each tab, record what you did and your result.

  1. Set up tabs: Add sheets named Task 1, Task 2, Task 3 …
  2. Add a header row (copy this to every tab):
    • Stage / Task
    • Goal (What to find/do)
    • Your SQL Query
    • Result (What appeared)
    • Proof (Paste snip or link)
    • Errors / Fixes
    • Time Taken (mins)
    • Next Step / Reflection
  3. Record evidence: After each stage, paste a Snipping Tool (or macOS screenshot) image into the tab or insert a link to the file (e.g., OneDrive/Drive link) in the Proof column.
  4. Be consistent: One stage per tab. Keep your SQL exactly as you typed it (don’t paraphrase).
  5. Save as: SQL_Island_Progress_YourName.xlsx (or Google Sheets).

Optional quality-of-life tips (for Google Sheets/Excel):

  • Data Validation: Make Stage a dropdown (Task 1–10) if you prefer one-sheet method.
