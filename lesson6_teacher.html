Teacher Guidance – Lesson 6
Spreadsheets (IT) vs Databases (CS) – SQL Island

Overview

Approx. length: 60 minutes Year group: 9 Options Taster

Focus: CS — introduce SQL via SQL Island (SELECT, WHERE, ORDER BY, LIKE). IT — record progress using a spreadsheet with one tab per stage, evidenced with screenshots.

Learning Objectives

  • Explain how databases store data (tables, records, fields, keys).
  • Use core SQL clauses: SELECT, WHERE, ORDER BY, AND/OR, LIKE.
  • Capture evidence neatly in a spreadsheet (one tab per stage) using Snipping Tool.

Lesson Flow (approx.)

Students complete the Do Now silently in Microsoft OneNote (Class Notebook) under “Lesson 6 – Do Now”. Throughout the lesson, pupils must use the Snipping Tool (Win + Shift + S) to evidence both CS and IT tasks in OneNote.
Approx. Time Stage Teacher Guidance (modelling optional) Student Outcome / Evidence
Start (≈ 5 min) Do Now – Silent retrieval Display the Do Now. Remind: quiet answers in OneNote; reveal briefly and move on. Responses saved under “Lesson 6 – Do Now”.
≈ 5 – 12 min Knowledge recap (DB basics) Run through table/record/field, primary key, data types; menu analogy (McDonald’s/Nando’s). Class can state what SQL does and why databases beat spreadsheets for related tables.
≈ 12 – 15 min CS setup – SQL Island Open SQL Island. If helpful, model one simple query:
SELECT name, age FROM citizens WHERE age >= 18 ORDER BY age DESC;		 Students ready to attempt stages independently.
≈ 15 – 35 min CS – SQL Island stages Circulate and prompt: “What are you selecting? What are you filtering? How are you ordering?” Encourage AND/OR, LIKE, tidy column lists, and testing iteratively. 6–8 stages completed; list of queries attempted. Evidence: Snip a working query + result and paste in OneNote under “Lesson 6 – CS Activity”.
≈ 35 – 52 min IT – Spreadsheet evidence pack Students create a spreadsheet SQL_Island_Name. If helpful, model making a new tab per stage and pasting a screenshot. Spreadsheet with tabs Task 1, Task 2, … each containing a clear screenshot (query + result). Evidence: Snip the spreadsheet structure into OneNote under “Lesson 6 – IT Activity”.
≈ 52 – 58 min Key Vocabulary (quick reinforce) Revisit: Record, Field, Query, SELECT, WHERE, ORDER BY, LIKE, Primary Key. Verbal recall; optional short note in OneNote.
Final minutes Exit Ticket Use the three questions from the student page. Responses in OneNote. Saved under “Lesson 6 – Exit Ticket”.

Support & Extension

  • Support: Provide a mini SQL crib sheet (SELECT col list; WHERE field op value; ORDER BY col ASC/DESC). Offer starter queries students can adapt. Allow pair programming.
  • Extension: Add COUNT(*) and GROUP BY where the stage allows; try a compound ORDER BY with two columns; write one query and an equivalent using LIKE.
  • SEND: Large fonts/high contrast; chunk tasks; modelling used only if needed; ensure independent use of Snipping Tool for evidence.
